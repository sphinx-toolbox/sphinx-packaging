PEP 503#normalized-names

The name is normalized per PEP 503

version must be a valid per PEP 440

requires-python maps to Requires-Python

The project’s description may have multiple lines.

A wheel’s supported platforms specify the OS and CPU for which the binary distribution was compiled.

Values in TOML must be one of the following types:

  • String

  • Integer

  • Float

  • Boolean

  • Offset Date-Time

  • Local Date-Time

  • Local Date

  • Local Time

  • Array

  • Inline Table

Inline Tables must be on a single line.

Unlike in YAML, a string must be surrounded by quotation marks.

You can place multiple objects in an array .

The following xrefs are not indexed: Float array .

PEP 621 defines the following fields in the [project] table:

  • name

  • version

  • description

  • readme

  • requires-python

  • license

  • authors

  • maintainers

  • keywords

  • classifiers

  • urls

  • scripts

  • gui-scripts

  • entry-points

  • dependencies

  • optional-dependencies

  • dynamic

authors and maintainers function the same; the difference is left up to the backend.
